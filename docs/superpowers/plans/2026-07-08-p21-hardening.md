### Task 2: Self-healing marks on export

**Files:**
- Modify: `crates/gitio/src/export.rs` (memo-seeding loop at ~line 378; `ExportReport` gains `stale_marks: usize`; the ~8 existing `ExportOptions` test literals need no change — only the report struct grows)
- Modify: `crates/cli/src/main.rs` (`run_push_git` + `run_export` print the stderr note when `report.stale_marks > 0`)

**Interfaces:**
- Produces: `ExportReport.stale_marks: usize` — marks whose git commit no longer exists in the target; those sc-ids were re-synthesized (and thus also appear in `new_marks` with fresh oids).

- [ ] **Step 1: Failing test** (export.rs tests, following the existing round-trip tests' setup — see the test at ~752 which already builds a `known` marks map):

```rust
#[test]
fn stale_mark_is_reverified_and_resynthesized() {
    // Export a 1-commit history to a bare target; capture new_marks.
    // Then corrupt the mark: replace its git oid with a valid-hex oid that
    // does NOT exist in the target (e.g. "aa" repeated 20 times).
    // Re-export with that marks map: must NOT error, report.stale_marks == 1,
    // the commit is re-synthesized (new_marks contains the sc id again),
    // and `git log`-equivalent (read_ref + walking) shows a valid history —
    // no broken parent chain.
}
```

- [ ] **Step 2: Implement.** In the seeding loop, verify existence before trusting a mark:

```rust
// Seed with commits already on the target (marks map). Verify each still
// EXISTS there before reuse: `git gc` in the target can prune commits a
// stale mark still names, and blindly reusing a pruned oid writes a broken
// parent chain (P18 follow-on, ADR-0031). A missing one is treated as
// unknown — the walk below re-synthesizes it — and counted.
let mut stale_marks = 0usize;
for (sc_id, git_hex) in opts.known_git_commits {
    let g = gix::ObjectId::from_hex(git_hex.as_bytes())
        .with_context(|| format!("bad git oid in marks for {sc_id}"))?;
    if /* object exists in target */ {
        commit_memo.insert(*sc_id, g);
    } else {
        stale_marks += 1;
    }
}
```

The existence check uses whatever `GitTarget` already exposes for reads (grep `GitTarget`'s impl in export.rs/import.rs — `find_object`/`contains` on the gix handle; do NOT add a second repo-open). Thread `stale_marks` into the report. CLI: after the existing push/export report printing, `if report.stale_marks > 0 { eprintln!("  note: {} mark(s) referenced git commit(s) pruned from the target; re-synthesized with fresh ids", report.stale_marks); }` in both `run_push_git` and `run_export`.

- [ ] **Step 3: Run** `cargo test -p scl-gitio` + `cargo test -p scl-cli network` (the P18 e2e tests must be undisturbed) + `cargo test` → green. **Step 4: Commit** — `git commit -am "fix(gitio,cli): export verifies marks against the target before reuse — pruned git commits re-synthesize instead of writing broken parent chains (P21)"`
